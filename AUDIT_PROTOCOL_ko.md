# 독립 원자료 감사 프로토콜

## 1. 목적

이 문서는 관내사전투표 동일 득표쌍 반복을 법적 결론으로 단정하기 위한 문서가 아니다. 목적은 공개자료에서 확인된 통계적 이상치를 원자료 감사 질문으로 바꾸는 것이다. 즉, 어떤 원자료가 공개되면 우연, 표시 오류, 입력 오류, 재확인표 처리, 또는 다른 행정적 원인을 구분할 수 있는지를 사전에 정한다.

감사의 핵심 질문은 다음과 같다.

1. 중앙선거관리위원회 선거통계시스템 공식 화면값과 원 개표상황표가 일치하는가.
2. 1차 투표지분류기 결과와 최종 공표값 사이의 차이가 개표상황표와 재확인표 기록으로 설명되는가.
3. 광주전남 5쌍과 송도 사례의 최종 동일 득표쌍이 원자료 단계에서도 같은 방식으로 재현되는가.
4. 동일 득표쌍 반복이 공개 시스템 표시 오류, 자료 입력 오류, 사후 정정, 재확인표 배분, 또는 기타 합법적 절차로 설명되는가.
5. 설명되지 않는 불일치가 남는다면 그 위치가 어느 개표단위, 어느 후보, 어느 처리 단계인지 특정할 수 있는가.

## 2. 감사 범위

최소 감사 범위는 다음 사건행이다.

- 광주전남 시·도지사선거 관내사전투표 동일 득표쌍 5쌍, 총 10개 개표단위 행
- 인천 연수구 송도1동·송도2동 관내사전투표 동일 득표쌍 1쌍, 총 2개 개표단위 행
- 같은 선거구 안의 비교 기준 행: 광주전남 전체 관내사전 개표단위 393개, 연수구 관내사전 개표단위 15개

확장 감사 범위는 다음을 포함한다.

- 동일 선거, 동일 후보 조합, 동일 관내사전투표 유형의 모든 읍·면·동급 개표단위 행
- 관내사전투표와 관외사전투표의 분리 결과
- 선거일 당일투표 결과와의 후보별 비교표
- 공식 통합 XLSX 개표파일, 선거통계시스템 HTML, 개표상황표 원본의 세 자료 간 대조

## 3. 필요한 원자료

독립 감사에는 최소한 다음 자료가 필요하다.

| 자료 | 필요한 이유 |
| --- | --- |
| 개표상황표 원본 | 최종 공표값이 현장 원표와 일치하는지 확인 |
| 1차 투표지분류기 후보별 결과 | 최종 동일 득표쌍이 처음부터 있었는지, 이후 단계에서 생겼는지 확인 |
| 재확인 대상 투표지 수 | 1차 결과와 최종 결과의 차이를 설명하는 후보별 재분류 규모 확인 |
| 재확인표 후보별 배분 내역 | 송도처럼 최종 동일성이 재확인 단계에서 형성됐는지 검증 |
| 무효표, 기권표, 제3후보 득표수 | 1·2위 득표쌍만 같은데 주변 수량이 다른지 확인 |
| 개표부서·심사집계부 로그 | 후보별 증감이 어느 처리 단계에서 발생했는지 확인 |
| 전산 입력·수정·공표 로그 | 입력 오류, 정정, 재공표 가능성 확인 |
| 분류기 식별자와 배치 식별자 | 같은 장비·같은 배치에서 반복이 집중됐는지 확인 |
| 참관인 이의제기 기록 | 현장 단계에서 문제 제기가 있었는지 확인 |
| 파일 해시와 생성 시각 | 감사 자료의 사후 변경 가능성 통제 |

개인 유권자 식별정보는 필요하지 않다. 감사는 후보별 집계표, 배치별 집계, 로그, 원표 대조만으로 수행되어야 한다.

## 4. 감사 절차

### 4.1 사건 정의 고정

감사 시작 전 다음 정의를 고정한다.

- 비교 단위: 관내사전투표 개표단위별 결과
- 후보 기준: 해당 선거에서 실제 1·2위 후보
- 동일 득표쌍: 두 개표단위가 1위 후보 득표수와 2위 후보 득표수를 동시에 공유하는 경우
- 광주전남 사건: 같은 후보 조합과 같은 투표 유형 안에서 5쌍 반복
- 송도 사건: 송도1동과 송도2동의 최종 1·2위 득표쌍 동일

이 정의는 감사 도중 바꾸지 않는다. 정의를 바꾸면 기존 확률값과 사건 수가 달라지므로 별도 분석으로 분리한다.

### 4.2 파일 보존과 해시

감사 대상 파일은 수령 즉시 다음 항목을 기록한다.

1. 파일명
2. 제공 기관
3. 제공 일시
4. 파일 크기
5. SHA-256 해시
6. 원본 여부와 사본 생성 경로

이후 모든 분석은 해시가 기록된 읽기 전용 사본에서 수행한다.

### 4.3 공식 화면값과 통합 파일 대조

다음 세 자료를 행 단위로 맞춘다.

1. 중앙선거관리위원회 선거통계시스템 HTML 화면값
2. 공식 통합 XLSX 또는 CSV 개표파일
3. 개표상황표 원본

대조 키는 선거명, 선거구, 구·시·군, 읍·면·동, 투표 유형, 후보명이다. 후보명 표기 차이가 있으면 후보자 ID 또는 정당·후보자 번호로 보조 매칭한다.

### 4.4 1차 결과와 최종 결과의 차이 재계산

각 사건행에 대해 다음 값을 계산한다.

\[
\Delta_i = F_i - S_i
\]

여기서 \(S_i\)는 후보 \(i\)의 1차 투표지분류기 결과이고, \(F_i\)는 최종 공표값이다. 모든 후보에 대해 \(\sum_i \Delta_i\)가 재확인 대상 투표지 수와 일치해야 한다.

일치하지 않으면 다음 중 하나가 필요하다.

- 별도 수개표 추가분
- 무효표에서 유효표로의 전환
- 유효표에서 무효표로의 전환
- 정정 공표 이력
- 입력 오류 수정 기록

### 4.5 재확인표 후보별 배분 검증

최종 동일 득표쌍이 재확인 단계에서 형성된 경우, 감사의 핵심은 재확인표의 후보별 배분이다. 각 개표단위별로 다음 표를 만든다.

| 개표단위 | 후보 | 1차 결과 | 재확인 증감 | 최종 결과 | 원자료 일치 여부 |
| --- | ---: | ---: | ---: | ---: | --- |
| 사건행 A | 후보 1 |  |  |  |  |
| 사건행 A | 후보 2 |  |  |  |  |
| 사건행 B | 후보 1 |  |  |  |  |
| 사건행 B | 후보 2 |  |  |  |  |

송도 사례처럼 1차 결과가 서로 달랐는데 최종 결과가 같아졌다면, 같은 최종값이 만들어진 후보별 증감 경로가 문서로 확인되어야 한다.

### 4.6 동일 득표쌍 재계산

원자료 대조가 끝난 뒤 동일한 스크립트 규칙으로 사건 수를 다시 계산한다.

1. 관내사전투표만 필터링한다.
2. 합계·소계 행을 제외한다.
3. 실제 1·2위 후보의 득표쌍을 만든다.
4. 같은 선거구 안에서 동일 득표쌍 그룹을 찾는다.
5. 광주전남 5쌍과 송도 1쌍이 그대로 남는지 확인한다.

공식 통합 파일에서 사건 수가 달라지면 그 차이의 원인을 행 단위로 기록한다.

## 5. 설명가설별 판정 기준

| 설명가설 | 확인되면 이상치가 약해지는 증거 | 확인되면 이상치가 강해지는 증거 |
| --- | --- | --- |
| 공개 화면 표시 오류 | 원 개표상황표와 통합 파일에서는 동일쌍이 사라짐 | 원표, 통합 파일, HTML이 모두 같은 동일쌍을 보임 |
| 단순 입력 오류 | 입력 로그에 정정 전후 값과 승인자가 남아 있음 | 정정 로그 없이 최종값만 동일쌍으로 남아 있음 |
| 재확인표 처리 | 1차 결과와 최종 결과 차이가 재확인표 후보별 배분으로 정확히 설명됨 | 재확인표 배분표가 없거나 합계가 맞지 않음 |
| 낮은 확률의 우연 | 주변 후보, 무효표, 기권표, 배치 로그가 모두 독립적으로 자연스러움 | 여러 쌍에서 같은 후보 방향·같은 처리 단계의 증감이 반복됨 |
| 사후탐색 효과 | 동일 기준으로 전국을 재탐색하면 유사한 반복이 자주 발견됨 | 같은 기준의 과거·전국 재탐색에서도 5쌍 이상이 드묾 |
| 행정 또는 개표 절차 이상 | 절차 로그가 완결되고 후보별 증감이 모두 설명됨 | 장비·배치·시간대·심사 단계에서 반복 패턴이 집중됨 |

## 6. 반증 기준

본 연구의 감사 요구는 다음 결과가 나오면 약화된다.

1. 공식 통합 개표파일 또는 원 개표상황표에서 2026년 사건행 값이 현재 공식 HTML 캐시와 다르고, 동일 득표쌍 반복이 사라지는 경우
2. 1차 투표지분류기 결과부터 최종 공표값까지의 후보별 증감이 모든 사건행에서 재확인표 기록으로 완전히 설명되는 경우
3. 동일한 사건 정의로 더 넓은 공식 원자료를 재분석했을 때 과거 또는 동시대 비교 선거에서 5쌍 이상 반복이 흔하게 관찰되는 경우
4. 지역 규모, 사전투표율, 후보 지지율, 도시·농촌, 연령 구조를 반영한 계층모형에서 광주전남 5쌍이 일반적인 예측 범위 안에 들어오는 경우

반대로 다음 결과가 나오면 감사 요구는 강해진다.

1. 원 개표상황표, 통합 파일, 공식 HTML이 모두 같은 동일 득표쌍을 보이는 경우
2. 1차 결과는 서로 달랐는데 재확인 또는 최종 입력 단계에서 동일쌍이 만들어진 경우
3. 재확인표 후보별 배분표가 없거나, 총량과 후보별 증감이 맞지 않는 경우
4. 같은 장비, 같은 시간대, 같은 심사·집계부, 같은 입력자 또는 같은 배치에서 반복이 집중되는 경우
5. 독립 재분석에서도 과거 기준선과 재표본추출 결과가 본 논문의 확률 결론을 유지하는 경우

## 7. 감사 산출물

감사 결과는 최소한 다음 파일로 공개되어야 한다.

- 사건행별 원자료 대조표
- 1차 결과와 최종 결과 후보별 증감표
- 재확인표 후보별 배분표
- 공식 HTML, 통합 파일, 개표상황표의 불일치 목록
- 동일 득표쌍 재계산 결과 CSV
- 분석 스크립트와 실행 로그
- 감사 자료 파일 해시 목록
- 설명가설별 판정표

## 8. 연구윤리와 공개 원칙

감사는 개인 유권자의 선택을 추적하지 않는다. 필요한 것은 후보별 집계, 개표단위별 집계, 장비·배치·시간대 로그, 공표값의 변경 이력이다. 개인정보가 포함된 내부 문서는 유권자 식별정보를 제거한 뒤 공개해야 한다.

감사자는 특정 결론을 미리 정하지 않아야 한다. 원자료가 우연 또는 행정절차로 설명하면 그 결론을 받아들여야 하고, 원자료가 설명하지 못하면 그 불일치의 위치와 크기를 공개해야 한다. 이 프로토콜의 목적은 통계적 이상치를 정치적 주장으로 확대하는 것이 아니라, 원자료로 설명 가능한 부분과 설명되지 않는 부분을 분리하는 것이다.
