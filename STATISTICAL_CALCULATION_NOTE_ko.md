# 통계 계산 노트

## 목적

이 문서는 원고의 핵심 확률값이 어떤 공식, 어떤 입력값, 어떤 스크립트에서 나왔는지 연결한다. 원고 본문은 해석을 담당하고, 이 노트는 계산 추적성을 담당한다.

## 1. 동일 득표쌍 수의 포아송 근사

한 선거구 안에서 비교 가능한 관내사전 개표단위 수를 \(N\), 가능한 1·2위 후보 득표쌍의 실질적 범위를 \(K\)라고 둔다. 두 개표단위가 같은 득표쌍을 가질 기대 충돌 수는 다음처럼 근사한다.

\[
\lambda = \frac{N(N-1)}{2K}
\]

동일 득표쌍 반복 수 \(C\)가 드문 충돌 사건으로 볼 수 있으면, \(C\)는 평균 \(\lambda\)의 포아송 분포로 근사할 수 있다.

\[
P(C \ge m) = 1 - e^{-\lambda}\sum_{c=0}^{m-1}\frac{\lambda^c}{c!}
\]

본 패키지의 기본값은 다음과 같다. 여기서 \(N\)은 보도된 사건행 수가 아니라, 광주전남 시·도지사 관내사전투표 안에서 서로 비교될 수 있었던 잠재 개표단위 수이다. 본 패키지는 중앙선거관리위원회 선거통계시스템 VCCP08 공식 HTML에서 광주광역시 5개 구와 전라남도 22개 시군을 모두 파싱했고, `관내사전투표`이면서 `합계`가 아닌 행을 광주 96개, 전남 297개, 합계 393개로 확인했다. 따라서 기본 계산은 \(N=393\)을 사용하고, \(N=350\)부터 \(N=450\)까지 민감도 분석을 병행한다.

| 기호 | 값 | 출처 |
| --- | ---: | --- |
| \(N\) | 393 | 선관위 VCCP08 공식 HTML에서 확인한 광주 96개 + 전남 297개 관내사전 개표단위 |
| \(K\) | 100,944.8 | 과거 시·도지사 기준선의 비교쌍 1,514,172개와 동일 득표쌍 15개에서 역산한 실질 득표쌍 범위 |
| \(\lambda\) | 0.7630705098231905 | `scripts/probability_sensitivity.py` |

\(K\)는 다음처럼 산출된다.

\[
\hat{K} = \frac{\text{과거 실제 1·2위 비교쌍 수}}{\text{과거 동일 득표쌍 수}}
       = \frac{1,514,172}{15}
       \approx 100,944.8
\]

위 값을 대입하면 다음 결과가 나온다.

| 기준 | 확률 | 확인 파일 |
| --- | ---: | --- |
| \(P(C \ge 3)\) | 0.04226077201454281 | `outputs/probability_core.csv` |
| \(P(C \ge 4)\) | 0.007734837111887716 | `outputs/probability_core.csv` |
| \(P(C \ge 5)\) | 0.0011484064248148407 | `outputs/probability_core.csv` |
| \(P(C \ge 6)\) | 0.00014322422035484283 | `outputs/probability_core.csv` |

실행 명령은 다음과 같다.

```bash
python3 scripts/probability_sensitivity.py
```

## 2. \(K\)와 \(N\) 민감도

포아송 근사는 \(K\)와 \(N\)의 선택에 영향을 받는다. 그래서 본 패키지는 두 민감도 파일을 별도로 생성한다.

| 민감도 | 출력 파일 | 설명 |
| --- | --- | --- |
| \(K\) 변화 | `outputs/probability_k_sensitivity.csv` | \(N=393\)을 고정하고 \(K\)를 50,000에서 500,000까지 변경 |
| \(N\) 변화 | `outputs/probability_n_sensitivity.csv` | \(K=100,944.8\)을 고정하고 \(N\)을 350에서 450까지 변경 |

이 민감도 분석의 역할은 단일 확률값을 절대화하지 않는 것이다. 특히 \(K=50,000\)처럼 가능한 득표쌍 범위를 작게 잡는 보수적 설정에서도 \(P(C \ge 5)\)는 약 2.05%로, 흔한 배경사건으로 바뀌지는 않는다.

## 3. 과거 실제 득표쌍 풀의 비복원 재표본추출

포아송 근사가 모형 가정을 포함하므로, 별도 경험적 검정으로 과거 실제 득표쌍 풀을 직접 재표본추출했다.

절차는 `scripts/bootstrap_governor_duplicates.py`에 구현되어 있다.

1. 2014·2018·2022년 시·도지사 관내사전투표에서 실제 1·2위 후보 득표쌍을 만든다.
2. 전체 10,322개 득표쌍 중 393개를 비복원으로 뽑는다.
3. 표본 안에서 동일 득표쌍 그룹 수를 센다.
4. 고정 난수시드 `20260609`로 200,000회 반복한다.

결과는 다음과 같다.

| 기준 | 초과 횟수 | 경험적 비율 | 보수적 보정 |
| --- | ---: | ---: | ---: |
| \(C \ge 3\) | 77/200,000 | 0.000385 | plus-one 0.0003899981 |
| \(C \ge 4\) | 4/200,000 | 0.00002 | plus-one 0.0000249999 |
| \(C \ge 5\) | 0/200,000 | 0 | plus-one 0.00000499998 |

\(C \ge 5\)가 0회였다는 말은 실제 확률이 0이라는 뜻이 아니다. 관측 해상도는 \(1/200{,}000=0.000005\)이고, 3의 법칙을 적용한 95% 상한은 \(3/200{,}000=0.000015\)이다.

확인 파일은 다음과 같다.

- `outputs/governor_bootstrap_summary.csv`
- `outputs/governor_bootstrap_histogram.csv`

실행 명령은 다음과 같다.

```bash
python3 scripts/bootstrap_governor_duplicates.py
```

## 4. 결론에서 허용되는 해석

위 계산은 다음 명제를 지지한다.

> 과거 시·도지사 관내사전투표 기준선과 2026년 공식 화면값을 함께 볼 때, 광주전남 5쌍 반복은 우연가설로 설명하기 어려운 통계적 이상치이다.

그러나 위 계산만으로 다음 명제까지 확정하지는 않는다.

> 특정 행위자, 장비, 입력 절차가 고의로 결과를 만들었다.

이 두 문장을 분리하는 것이 본 원고의 핵심 방어선이다. 계산은 감사 필요성을 지지하지만, 원인 확정에는 개표상황표 원본, 1차 분류기 결과, 재확인표 배분 내역, 전산 입력 로그가 필요하다.
