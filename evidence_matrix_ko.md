# 증거 대응표

## 목적

이 문서는 원고의 핵심 주장, 해당 주장을 뒷받침하는 재현 산출물, 아직 추가 공개가 필요한 원자료를 한눈에 분리한다. 학회 제출 또는 사전 검토 과정에서 "어떤 주장이 어떤 파일로 확인되는가"를 빠르게 대조하기 위한 보조 문서이다.

## 현재 자료만으로 확정하지 않는 주장

| 주장 | 현재 상태 | 필요한 추가 자료 |
| --- | --- | --- |
| 특정 행위자 또는 기관이 고의로 득표값을 만들었다. | 현재 자료만으로 확정하지 않음 | 수사·감사 자료, 입력자·시간대 로그, 개표상황표 원본 |
| 동일 득표쌍이 재확인표 배분 단계에서 만들어졌다. | 송도 사례에서는 검증 요구가 강하지만, 광주전남 5쌍에는 원자료가 필요함 | 1차 분류기 결과, 재확인 대상 투표지 후보별 배분 내역 |
| 전산 시스템 표시 오류가 원인이다. | 가능 원인 중 하나지만 확인되지 않음 | 공개 시스템 DB 원장, 개표상황표 원본, 로그 대조 |
| 유권자 자기선택만으로 사전-당일 편차가 전부 설명된다. | 가능하지만 현재 원고의 단순 부호검정만으로는 확정 불가 | 연령·성별·지역·정당별 사전투표 독려 자료, 관내·관외 분리 분석 |

## 원자료 감사 판정표

| 원자료 | 우연 설명을 약화시키는 확인 결과 | 우연 설명을 회복시키는 확인 결과 |
| --- | --- | --- |
| 개표상황표 원본 | 원본도 12개 사건행과 6개 동일쌍을 보인다. | 원본값과 화면값이 달라 동일쌍이 사라진다. |
| 1차 분류기 결과 | 1차 결과는 달랐는데 재확인·합산 뒤 동일쌍이 생긴다. | 1차부터 최종까지 동일값이 독립적으로 일관된다. |
| 재확인표 후보별 배분 | 여러 쌍에서 같은 후보 방향의 보정이 반복된다. | 재확인표 배분이 동일쌍 생성과 무관하다. |
| 전산 입력 로그 | 동일쌍 관련 행에서 비정상적 수정·재입력 패턴이 집중된다. | 입력 로그가 시간순으로 일관되고 수정 흔적이 없다. |
| 관내·관외 분리자료 | 동일 후보 조합의 동일쌍이 관내사전투표에 집중된다. | 관내·관외·당일 자료에서 같은 정도의 반복이 나타난다. |
| 투표함·개표배치 인수인계 기록 | 동일쌍 관련 배치가 같은 장비·시간대·심사 흐름에 집중된다. | 배치 이동과 처리 흐름이 독립적이고 시간순으로 설명된다. |

## 제출용 결론 문장

현재 패키지가 가장 강하게 방어할 수 있는 결론은 다음이다.

> 중앙선거관리위원회 선거통계시스템 공식 화면값과 과거 공식자료 기준선을 함께 볼 때, 2026년 광주전남 5쌍 동일 득표 반복은 우연가설과 양립하기 어려운 통계적 이상치이며, 개표상황표 원본과 1차 분류기 결과의 독립 검증 없이는 통계적으로 종결될 수 없다.

이 결론은 특정 원인을 법적으로 확정하는 문장이 아니라, 공개 원자료 감사 필요성을 통계적으로 제시하는 문장이다.
